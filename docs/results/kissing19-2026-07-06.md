# Kernel-attested record kissing-number bound k(19) ≥ 11948 (Boon Suan Ho 2026)

**Date:** 2026-07-06 · **Track:** T5 (external audit) / T9 (external corpus) · **Tier:** audit ·
**EV:** verification-amplification · **Gate:** GREEN (exact bit arithmetic; Lean 4.31 `decide`)

## What this is

An independent, exact-arithmetic confirmation of a **current record** in a domain new to the ledger (**sphere
packing / kissing numbers**). The kissing number `k(n)` is the maximum number of unit spheres that can touch a
central unit sphere in `n` dimensions. Boon Suan Ho ([arXiv:2603.10425](https://arxiv.org/abs/2603.10425), 2026)
proves

> **k(19) ≥ 11948**,

improving the Cohn–Li bound `k(19) ≥ 11692` by **256** — the best known lower bound in dimension 19.

By the **Cohn–Li odd-sign construction**, `k(19) ≥ 10668 + |A|` for any binary code `A` of length 19 and
**minimum distance ≥ 5** contained in a fixed **5-punctured extended binary Golay code** `D` (each such codeword
contributes one extra kissing vector). Cohn–Li used a linear code of size 1024 (→ 11692); Ho constructs a
**nonlinear** code of size **1280** (→ 11948).

## Fully reconstructible — no data file needed

The construction is explicit and reconstructible from ~14 small subsets of `{1,…,19}` (words are supports;
addition is symmetric difference). With coordinates as 19-bit masks:

- `D = span(m₁..m₆, s₁..s₄, r₁, r₂)` — dim 12, `|D| = 4096`, a 5-punctured extended Golay code;
- `M = span(m₁..m₆)` (dim 6), `K = span(M, s₁..s₄)` (dim 10);
- `B = (s₁+M) ∪ (s₂+M) ∪ (s₃+M) ∪ (s₄+M) ∪ (s₅+M)` with `s₅ = {3,5,7,10}` — `|B| = 5·64 = 320`;
- `A = B ∪ (B+r₁) ∪ (B+r₂) ∪ (B+r₁+r₂)` — `|A| = 4·320 = 1280`.

Leibniz reconstructs `A` from the generators (rather than reading the author's 726 KB vector file) and verifies
by exact bit arithmetic:

- `dim M/K/D = 6/10/12`; `|B| = 320`; `|A| = 1280`; `A ⊆ D`; the identity `s₅ = s₁+s₂+s₃+s₄+m₄+m₆`;
- `D` has minimum nonzero weight 3, and its **21 words of weight 3 or 4 are exactly the paper's Table 1**;
- **minimum distance exactly 5**, two independent ways: the full `C(1280,2) = 818560`-pair census gives min
  distance 5, and the **forbidden-difference test** (no two codewords differ by one of the 21 weight-3/4 words)
  agrees;
- hence **`k(19) ≥ 10668 + 1280 = 11948`**.

### A faithfulness finding

Reproducing the paper's **Table 1** exactly against the computed weight-3/4 words of `D` caught **one
transcription error** in our reading of the table (a coordinate `19` that should be `15`); the exact set is
authoritative, and after correction all 21 words match. This is precisely the kind of check the census is for.

## The Lean kernel re-decides the finite core

The **Lean 4.31 kernel** independently re-decides (plain `decide`, report-only) in
[`docs/crt/kissing19.lean`](../crt/kissing19.lean), with everything rebuilt inside the kernel — `D` is regenerated
from the 12 generators by subset-XOR, and membership in `A` uses a balanced binary search tree for `O(log|A|)`
lookups:

- `kissing_bound` — `10668 + 1280 = 11948`;
- `kissing_distinct` — `A` is strictly increasing with 1280 entries (1280 distinct codewords);
- `kissing_subset_D` — every codeword is in `D` (parity check against a basis of `D^⊥`);
- `kissing_forbidden_complete` — the weight-3/4 words of the **rebuilt** `D` are exactly the 21 forbidden words;
- `kissing_mindist` — no codeword plus a forbidden word lands back in `A` (so, since `A ⊆ D` has min weight 3
  and every weight-3/4 difference is a forbidden word, `A` has **minimum distance ≥ 5**);
- `kissing_negcontrol` — a codeword `a₀` and a forbidden word `s₀` give `a₀+s₀` at distance ≤ 4 from `a₀` but
  **not** in `A` — a witnessed distance-≤4 neighbour correctly excluded (a discriminating negative control).

`#print axioms` shows **at most `[propext]`** (a canonical trusted axiom) for every theorem — no `native_decide`,
no `sorry`, no `sorryAx`.

## Honest scope

This certifies the **finite coding-theoretic core** — that Ho's explicit 1280-word code sits inside a 5-punctured
extended Golay code with minimum distance 5 — which, via the Cohn–Li odd-sign construction (cited, not re-proved
here), yields `k(19) ≥ 11948`. What is **not** re-verified: the geometry of the Cohn–Li base configuration
(`10668` points) and the odd-sign lifting itself (external analytic input); and the identification of `D` with a
5-punctured *extended Golay* code is checked as a `[24,12,8]` doubly-even self-dual code (Prop 2) but the standard
uniqueness of that code is cited. The backend is **report-only**: nothing sets `kernel_verified` or touches
`trust.py`; `tests/test_invariants.py` is byte-identical.
