# Kernel-attested audit: the srg(1666,105,0,7) non-existence proof does not decide the case

**Cycle 35 · 2026-07-06 · Spectral / algebraic graph theory (strongly regular graphs) · kind: audit · report-only**

## Target

Belousova, V. I., Makhnev, A. A., & Tokbaeva, A. A. (2026). *A strongly regular graph with
parameters (1666, 105, 0, 7) does not exist.* Bulletin of Perm University. Mathematics.
Mechanics. Computer Science, 1(72), 29–34. DOI
[10.17072/1993-0550-2026-1-29-34](https://doi.org/10.17072/1993-0550-2026-1-29-34) (Russian).

The paper proves **Theorem 1** — the bipartite distance-regular graph with intersection array
`{105, 104, 98, 7, 1; 1, 7, 98, 104, 105}` does not exist — and concludes **Theorem 2**: the
strongly regular graph srg(1666, 105, 0, 7) does not exist (an srg exists iff its bipartite double
does). srg(1666,105,0,7) is the smallest (s=1) member of the feasible triangle-free family with
μ=7, k=49s²+49s+7, s∈{1,2,7} (Biggs, [arXiv:0911.2455](https://arxiv.org/abs/0911.2455)); it is a
long-standing **feasible-but-open** parameter set, beyond the range of every published table
(Brouwer's on-line SRG tables stop at v≤1300; Brouwer–Van Maldeghem 2022 at v≤512).

## What the daemon re-decided

Everything below is reconstructed **from first principles** (exact rational arithmetic) starting
from the intersection array — never from the paper's transcription — then the finite core is
re-decided by the Lean 4.31 kernel (`docs/crt/srg1666_audit.lean`, plain `decide`; `#print axioms`
shows at most `[propext]`).

**The paper's Lemmas 1–3 are correct.** The intersection numbers pⁱⱼₖ, the dual eigenmatrix Q, the
Krein parameters, and the triple intersection numbers all reproduce exactly — including that the
config d(u,v)=d(u,w)=2, d(v,w)=4 (Lemma 2) is **uniquely** determined and the all-distance-2 config
(Lemma 3) is a **one-parameter family** in r₁ ∈ {0,…,7}. One incidental typo: Lemma 1 prints
p²₃₃ = 543, but the correct value is **1461**, forced by the duality k₂·p²₃₃ = k₃·p³₂₃ and by the
row sum p²₃₁ + p²₃₃ + p²₃₅ = k₃ = 1560. Not load-bearing.

**The proof of Theorem 1 does not establish non-existence.** Its contradiction compares two
computations of the mean value λ̄ of the auxiliary graph Λ (the distance-2 graph on Γ₂(u), which
is p²₂₂ = 1461-regular on k₂ = 1560 vertices). Written with the correct arithmetic, **both routes
are identically 1999388/1461**:

| route | expression | value |
|---|---|---|
| edge-count (Lemma 2) | 1461·1460 − 98·1364 | 1999388 |
| f₄ double-count (Lemma 3) | 98·97 + 1461·1362 | 1999388 |

The paper's apparent gap (λ̄ = 1362.905 vs 1368.09) is an artifact of **two compensating
arithmetic errors**: it uses **104** for the true non-neighbour count 1560 − 1 − 1461 = **98**, and
divides by **1560** instead of by the degree **1461**. The structural reason the two routes must
agree is the identity

> [222]₍Lemma 2₎ + [224]₍Lemma 2₎ = 1364 + 97 = 1461 = p²₂₂,   so   S₁ − S₂ = 98·(1461 − 1364 − 97) = 0.

**The triple-intersection method leaves the array feasible.** Every metrically-valid base triple —
with all 81 zero-Krein equations imposed — has a non-negative integer solution; the all-distance-2
config has **exactly 8** solutions (r₁ = 0,…,7). The kernel verifies all 8 witnesses against every
marginal, every zero-Krein equation, and non-negativity, and rejects the r₁ = 8 witness (a
discriminating negative control).

## Independent corroboration (canonical tool)

The paper states it used **sage-drg** (Vidali). Running the canonical sage-drg on this exact array
in **SageMath 10.9 (native arm64)**:

- `DRGParameters([105,104,98,7,1],[1,7,98,104,105]).check_feasible()` → **clean** (not ruled out);
- `check_absoluteBound / check_combinatorial / check_2design / check_terwilliger /
  check_localEigenvalues / check_handshake` → **all pass**;
- `tripleSolution_generator` over all realizable distance-triples → **no zero-solution config**;
  the (2,2,2) config returns **8 solutions**; and it independently reports p²₃₃ = 1461;
- sage-drg on the antipodal quotient itself — srg(1666,105,0,7) as the diameter-2 array {105,104;1,7}
  — is also **clean** (not ruled out).

Independently, **SageMath's own strongly-regular-graph existence oracle** returns
`strongly_regular_graph(1666, 105, 0, 7, existence=True)` → **`Unknown`** (i.e. **open**), not
`False`. Sage 10.9 (May 2026) encodes the community's catalogue of constructions and non-existence
results, and it still classifies this case as open.

So the very tool the authors cite does **not** rule out the array, and the canonical SRG database
regards the case as unresolved. Three further independent from-scratch reconstructions
(A₁-recurrence + Gaussian elimination; blind intersection-matrix recurrence; spectral/idempotent
intersection numbers + exact simplex LP + Fourier–Motzkin) all agree: Lemmas 1–3 correct,
contradiction vacuous, array feasible.

## Verdict and scope (read this)

This is an **audit**, not an existence or non-existence claim. The kernel-attested conclusion is:

> The proof of Theorem 1 as published does **not** decide the parameter set. Its contradiction is
> arithmetically vacuous, and the triple-intersection method it relies on leaves the array feasible.
> Therefore srg(1666, 105, 0, 7) should be regarded as **open**, pending a corrected argument.

We make **no** claim that the graph exists or does not exist — only that this argument does not
settle it. The group's method (triple intersection numbers + Krein equality) is the correct and
standard tool for this class and they have applied it successfully elsewhere; it is entirely
possible a corrected argument recovers the theorem. Nothing here should be read as asserting the
graph exists.
